# Idea Log - 2026-03-16

## Problem Statement
We sometimes jot ideas but forget to note who should follow up and how the idea connects to the broader collaboration flow, so each log ends up sounding like a random brainstorm instead of a practical next step.

## Idea
- Create a “Debrief Card” section for each daily idea that:
  - Captures who (assistant, human, other stakeholders) should take the next concrete action.
  - Clarifies how the idea fits in the collaboration timeline (start, iterate, release).
  - Lists the minimal artifact we’d need to prove the idea is worth more effort (prototype snippet, checklist, user quote).

## UX & Testing Focus
- Keep each card short; use three bullet lines (Who, Timeline, Artifact) with bolded labels so you can skim and approve fast.
- Add a Vitest check that ensures every idea entry has a Debrief Card with non-empty values before the daily log is considered complete.
- Offer a tiny helper script (`npm run idea-card`) that prints the latest card in terminal-friendly blocks to aid standup reviews.

## Next Steps
1. Save today’s Debrief Card under this entry and leave it pending your review.
2. Once you give the green light, create `ideas-generated-ai/debrief-cards` with templates, tests, and helper scripts to generate new cards.
3. Watch how the cards flow—do they make approval faster? Do they reveal missing actions sooner? Adjust prompts based on what helps you decide.
